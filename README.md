# 🐺 人狼ドッチ - オンライン

3〜8人・5〜10分・司会者不要の正体隠匿パーティゲーム「人狼ドッチ」を、
ブラウザだけで友達とオンラインプレイできるWebアプリです。

- サーバー不要（GitHub Pages + PeerJS のP2P通信で完結）
- アカウント登録不要（ルームコードを共有するだけ）
- 役職は `data/roles.csv` を Excel で編集するだけで追加・調整可能

## 遊び方

1. ホストがサイトを開き、名前を入力して「部屋を作る」
2. 表示された **ルームコード（4文字）** を友達に伝える
3. 友達は名前とコードを入力して「部屋に参加」
4. 3人以上そろったらホストが「ゲーム開始」

議論はボイスチャット（Discord等）または対面での会話で行ってください（アプリ内にチャットはありません）。

## GitHub Pages で公開する手順

1. GitHub で新しいリポジトリを作成（例: `jinro-dodge`）
2. このフォルダの中身をすべて push する

   ```bash
   git init
   git add .
   git commit -m "人狼ドッチ 初版"
   git branch -M main
   git remote add origin https://github.com/<ユーザー名>/jinro-dodge.git
   git push -u origin main
   ```

3. リポジトリの **Settings > Pages** を開き、
   Source を「Deploy from a branch」、Branch を `main` / `(root)` にして Save
4. 数分後、`https://<ユーザー名>.github.io/jinro-dodge/` で遊べるようになります

## ローカルでの動作確認

> ⚠ **index.html をダブルクリックで直接開いても動きません**（ブラウザのセキュリティ制限のため）。
> 必ず以下のどちらかの方法で起動してください。

**方法1（かんたん）**: フォルダ内の **`ゲーム起動.bat`** をダブルクリック。
サーバが起動してブラウザが自動で開きます。黒い画面はゲーム中閉じないでください。

**方法2（手動）**:

```bash
# このフォルダで
python -m http.server 8000
# → ブラウザで http://localhost:8000 を開く
```

タブを3つ開いて、1つで部屋を作り、残り2つで参加すれば1人でテストできます。
※どちらの場合もオンライン対戦の通信（PeerJS）にはインターネット接続が必要です。

## 役職の追加方法（`data/roles.csv`）

`data/roles.csv` を Excel で開いて行を追加・編集するだけです。
保存形式は「CSV UTF-8（コンマ区切り）」を推奨（通常のCSV保存でも自動判別して読み込めます）。

| 列 | 説明 |
|---|---|
| `id` | 半角英数のユニークなID |
| `名前` | 画面に表示される役職名 |
| `陣営` | `市民` / `人狼` / `おばけ` のいずれか |
| `アクション種別` | 下表参照 |
| `実行フェーズ` | `明け方` / `昼過ぎ` / 空欄（アクションなし） |
| `人狼判定` | `1`=追放されたら「人狼」として扱う。狂人のような「人狼陣営だが人狼ではない」役職は`0` |
| `枚数_3人`〜`枚数_8人` | その人数のとき山札に入れる枚数 |
| `説明` | カードに表示される説明文 |
| `有効` | `1`で使用、`0`で除外 |

### 使えるアクション種別

| 種別 | 動き |
|---|---|
| `none` | 能力なし |
| `mate_check` | 明け方に同じ役職の仲間を確認（人狼タイプ） |
| `peek_used` | 対象1人の使用カードを見る（占い師タイプ） |
| `peek_field` | 対象1人の伏せカードを見る（警官タイプ） |
| `swap` | 対象1人の伏せカードと使用カードを交換。パス可（DJタイプ） |
| `peek_team` | 対象1人の陣営だけを知る。役職名は分からない（名探偵タイプ） |

昼過ぎのアクション順は **CSVの行順**（警官 → DJ → 名探偵）。
その人数で枚数0の役職のステップは自動的に省略されます（枚数表は公開情報のため）。

**枚数の制約**: 各人数の列の合計は「人数 × 配布枚数」（5〜8人は2枚、3〜4人は3枚）に
一致させてください。合わないとゲーム開始時にエラーメッセージが表示されます。

新しい動きの役職（例: カードを見てから交換できる「怪盗」など）を作りたい場合は、
`js/game/engine.js` にアクション種別のハンドラを1つ追加すれば対応できます。

## ルールブック

### 基本情報
- プレイ人数: 3〜8人 / 所要時間: 5〜10分 / 進行役: 不要

### 公式ルール（初期版）

**準備**: 各プレイヤーに役職カードを2枚配布。その中から1枚を選んで使用し、残りは場に伏せて置く。

**フェーズ進行**
1. **明け方**: 人狼→仲間確認 / 占い師→誰か1人のカードを見る
2. **昼（議論）**: 約3分間の話し合い
3. **昼過ぎ（役職アクション)**: 警官→誰かの場カード1枚を見る / DJ→他人1人の「場に伏せたカード」と「現在使っているカード」を交換（交換しない選択も可）
4. **夕方（投票）**: 約1分の最終議論後、投票で追放者を決定

**勝利条件**
- 追放者が人狼 → 市民陣営勝利
- 追放者が市民系 → 人狼陣営勝利
- 追放者がおばけ → おばけの単独勝利

### ハウスルール① ポイント制（本アプリでは常時有効）
- 市民陣営勝利 → 市民陣営に **+1点** / 人狼陣営勝利 → 人狼陣営に **+2点** / おばけ勝利 → **+3点**
- 市民陣営のプレイヤーが市民陣営に投票 → **-1点**
- 「もう一戦」で得点を持ち越して連戦できる（大会形式）

### ハウスルール② 少人数戦（3〜4人・自動適用）
- 各プレイヤーに役職カードを **3枚** 配布し、1枚使用・2枚伏せる
- 警官は対象の伏せカード **2枚とも** 確認できる
- 占い師は初日の占いを「しない」選択が可能。しなかった場合、昼過ぎのアクションは
  **1. 占い師 → 2. 警官 → 3. DJ** の順で実行

### アプリでの補完ルール（原ルールに規定がない部分・調整）
- **役職アクションは完全非公開**。誰が・誰に・何をしたかは一切表示されず、ログにも残らない
  （結果は実行者本人にだけ見える）。
- **DJの交換は交換された本人にも通知されない**。全カードは裏のまま進行し、
  自分の使用カードが入れ替わっていても結果発表まで分からない。
- **各役職アクションは一律15秒固定・短縮不可**。該当者がいなくても必ず15秒経過するため、
  タイミングや所要時間から役職を推測（メタ読み）することはできない。
- **占い師が明け方に時間切れ（または少人数戦で延期）した場合、昼過ぎに占える**。
  延期・時間切れの有無を隠すため、全人数で昼過ぎの先頭に占い師の時間が毎回設けられる。
- **夕方の最終議論は5分**（昼の議論は約3分）。
- **投票が同数の場合は決選投票を行わず、同数だった全員を追放する。**
  追放者の中に人狼が1人でもいれば市民陣営の勝利。おばけが含まれていればそのおばけは単独勝利(+3)が並立する。
- **投票には「人狼はいない」（誰も追放しない）の選択肢がある。**
  **全員一致のときだけ成立**し、1人でも別の投票をしたら「人狼はいない」票は無効票になる。
  成立した場合、使用カードに人狼が本当にいなければ市民陣営の勝利(+1)、
  潜んでいれば人狼陣営の勝利(+2)。「人狼はいない」への投票はペナルティ(-1)の対象外。
- **3人戦は人狼1枚＋狂人1枚**。狂人は人狼陣営（人狼勝利で+2）だが人狼ではないため、
  狂人を追放しても人狼陣営の勝利になる。狂人は誰が人狼か知らされない。
- **名探偵**（6人以上）: 昼過ぎの最後に誰か1人の**陣営だけ**を知る。役職名までは分からず、
  狂人も「人狼」陣営と表示される。
- **賞金稼ぎ**（7人以上）: 自分の投票先が人狼（人狼扱いの狂人含む）だったら追加で**+1点**。
  市民陣営なので通常の勝利点・投票ペナルティも適用される。
- 勝敗・ポイントの陣営判定は「追放時点の使用カード」（DJ交換後のカード）で行う。

### 画面の機能
- **ロビーに人数別の役職枚数表**を表示（現在の人数の列を強調）。ゲーム中も
  カード選択画面と手札パネルに**この山札の内訳**（例: 人狼×2・占い師×1…）が表示される。
- カード選択後も配られた2〜3枚は画面に表示され続け、**使用中／伏せ1／伏せ2** のラベル付きで
  各カードに**メモ**を書き込める。
- さらに**他のプレイヤー全員のカードも裏向きで表示**され、1枚ずつメモを書き込める
  （占い・警官の結果や推理の記録に）。
- 占い・確認・交換の対象は、全員のカード（使用中／伏せ）を並べたボードから**直接タップ**で選ぶ。
- **フェーズの切り替わり（タイマー終了）はチャイム音＋バイブで知らせる**（投票開始は強調）。
  ※ブラウザの仕様上、一度画面をタップ／クリックした後から音が鳴ります。
  ※バイブは対応端末（Android等）のみ。iPhone(iOS)はバイブAPI非対応のため音のみです。
- チャット機能はない。議論はボイスチャットや対面での会話で行う。

## ファイル構成

```
├── index.html            # 画面（1ページ構成）
├── css/style.css         # スタイル
├── js/
│   ├── main.js           # エントリポイント・画面遷移
│   ├── net/              # 通信（PeerJS）: host.js / client.js / protocol.js
│   ├── game/             # ゲームロジック: engine.js / deck.js / scoring.js / views.js

```

## しくみ（技術メモ）

- 通信は [PeerJS](https://peerjs.com/)（WebRTC）。ホストのブラウザがゲームサーバの役割を持ち、
  カード配布・アクション解決・勝敗判定をすべて行います。
- 各プレイヤーには「その人が見てよい情報」だけが送信されるため、
  通信を覗いても他人のカードは分かりません。
- ホストがページを閉じるとゲームは終了します（ホスト=サーバのため）。
